ide工程结构的几种方案

第一种 nuclei sdk

nuclei sdk的目录结构(OneOS示例工程为例)：

Application文件夹放应用层面的业务代码。

arch文件夹放OneOS移植到CM32M4xxr的相关文件。

board文件下放的是OneOS启动初始化相关的文件以及启动文件。

kernel文件夹下放的是OneOS内核相关的代码。

最外层包括OneOS的配置文件和工程的链接文件。

该文件夹下是CM32相关的文件

board下是CM32M433R_START实现led，log等

CM32M4xxr_Algo_Library文件夹是算法库

CM32M4xxr_stdPeriph_Diriver文件夹下是固件库

Env 文件夹是 系统一些函数的实现，为了某些程序可以移植，

NMSIS下Core 是cpu相关外设的接口。

Device下是设备驱动的配置文件和链接文件等

DSP和Libarry是和DSP相关的api。

第二种 MRS

core： cpu相关外设的函数

debug：调试相关的接口实现

FreeRTOS(或者其他rtos)：包括rtos内核以及移植适配相关的所有文件

Ld: 链接文件

Peripheral: 固件库相关文件

Startup: 启动文件

User: 应用程序相关文件包括rtos的配置文件

第三种 CSI SDK

  • csi_core： 定义cpu相关的紧耦合外设的接口规范

    ​ 包括中断控制器，DCC，寄存器等相关设备，和cpu的类型相关。

  • csi_driver：常用的驱动接口规范

    ​ 例如gpio，timer，具体实现和电路板相关。硬件相关的驱动实现，以及操作系统的移植框架。

  • csi_kernel：rtos统一接口规范

    ​ 包括rtos内核，不同的rtos通过adapter层，实现一套同一rtos接口，让上层组件可以调用统一的接口来实现功能，增加程序的可移植性。

  • board：板级初始化相关的接口定义

    ​ 包括时钟，串口的初始化和开启。

  • libs：相关库和组件

    ​ 例如，posix，libc，日志，ringbuffer，mm内存管理等组件库

  • project: 业务代码

    ​ 通过rtos内核统一的头文件，调用rtos接口
